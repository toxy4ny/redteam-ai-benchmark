# Code of Conduct

This project uses a concise code of conduct for project spaces: issues, pull requests, discussions, reviews, and other maintainer-run communication channels.

## Expected Behavior

- Be direct, respectful, and constructive.
- Critique ideas, code, tests, and claims rather than people.
- Assume contributors may have different backgrounds, languages, and threat-modeling experience.
- Keep offensive-security examples clearly tied to authorized research, labs, benchmarks, or defensive validation.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private data, credentials, non-public targets, or exploit details for systems outside an authorized scope.
- Repeatedly derailing maintainer decisions after a decision has been explained.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; or block participants when needed to keep the project usable and safe.

Report conduct problems privately through the repository owner's GitHub profile contact path. If that is unavailable, open a minimal public issue that asks for a private maintainer contact and does not include sensitive details.

This policy is inspired by the Contributor Covenant and tailored for this repository.
